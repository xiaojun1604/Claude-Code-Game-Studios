# 可用技能 (斜杠指令 / Slash Commands)

| 指令 | 用途 |
|---------|---------|
| `/start` | 首次上手引导 —— 询问您的位置，然后引导您进入正确的工作流 |
| `/design-review` | 评审游戏设计文档的完整性和一致性 |
| `/code-review` | 对文件或变更集进行架构级代码审查 |
| `/playtest-report` | 生成结构化的游戏测试报告模板 |
| `/balance-check` | 分析游戏平衡数据并标记异常值 |
| `/sprint-plan` | 生成或更新冲刺计划 (Sprint Plan) |
| `/bug-report` | 创建结构化的 Bug 报告 |
| `/architecture-decision` | 创建架构决策记录 (ADR) |
| `/asset-audit` | 审核资产的命名、尺寸和管线合规性 |
| `/milestone-review` | 回顾里程碑进度并生成状态报告 |
| `/onboard` | 为新贡献者或 Agent 生成入职上下文 |
| `/prototype` | 搭建临时原型以测试核心机制或技术方案 |
| `/release-checklist` | 为当前版本生成并验证发布前检查清单 |
| `/changelog` | 从 git 提交和冲刺数据中自动生成更新日志 (Changelog) |
| `/retrospective` | 进行结构化的冲刺或里程碑回顾 (Retrospective) |
| `/estimate` | 提供包含复杂度和风险分解的结构化工作量估算 |
| `/hotfix` | 紧急修复工作流，包含审计追踪，绕过正常冲刺流程 |
| `/tech-debt` | 扫描、跟踪、确定优先级并报告代码库中的技术债务 |
| `/scope-check` | 对比原始计划分析功能或冲刺范围，标记范围蔓延 (Scope Creep) |
| `/localize` | 本地化工作流：扫描硬编码字符串、提取并验证翻译 |
| `/perf-profile` | 结构化性能分析，识别瓶颈并提供改进建议 |
| `/project-stage-detect` | 自动分析项目状态、检测阶段、识别差距并建议后续步骤 |
| `/reverse-document` | 从现有实现生成设计或架构文档 (逆向工程) |
| `/team-combat` | 编排战斗团队：游戏设计师 + 玩法程序员 + AI程序员 + 技术美术 + 音效师 + QA测试 |
| `/team-narrative` | 编排叙事团队：叙事总监 + 编剧 + 世界观架构师 + 关卡设计师 |
| `/team-ui` | 编排 UI 团队：UX设计师 + UI程序员 + 艺术总监 |
| `/team-release` | 编排发布团队：发布经理 + QA负责人 + DevOps工程师 + 制作人 |
| `/team-polish` | 编排打磨团队：性能分析师 + 技术美术 + 音效师 + QA测试 |
| `/team-audio` | 编排音频团队：音频总监 + 音效师 + 技术美术 + 玩法程序员 |
| `/team-level` | 编排关卡团队：关卡设计师 + 叙事总监 + 世界观架构师 + 艺术总监 + 系统设计师 + QA测试 |
| `/launch-checklist` | 完成跨部门的全方位上线准备验证 |
| `/patch-notes` | 从 git 历史和内部数据生成面向玩家的补丁说明 (Patch Notes) |
| `/brainstorm` | 使用专业工作室方法进行引导式构思 (MDA, SDT, Bartle, 动词优先) |
| `/gate-check` | 验证开发阶段之间的晋级准备情况 (通过/疑虑/未通过) |
| `/map-systems` | 将游戏概念分解为系统，映射依赖关系，确定设计优先级，指导各系统的 GDD |
| `/design-system` | 引导式、分章节的单个游戏系统 GDD 编写，支持交叉引用和增量写作 |
| `/setup-engine` | 配置引擎与版本，检测知识差距，填充版本感知的参考文档 |
