---
description: 架构级代码审查
---
1. 确认用户希望审查的文件路径或最近的 Git 变更集。
2. 针对以下维度进行深度分析：
    - **架构一致性**：是否遵循了项目定义的模式（见 `coding-standards.md`）。
    - **性能瓶颈**：检查是否存在不必要的 GC 分配、过度频繁的 Update 调用等。
    - **可维护性**：变量命名、函数复杂度及文档注释。
    - **平台适配**：针对移动端或 PC 端的特定考虑。
3. 提供具体的修改建议，并在可能的情况下提供重构后的代码片段。
4. 总结审查结果，并给出改进建议的优先级。
